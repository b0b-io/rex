# Rex TUI - User Experience Design

## Design Philosophy

The Rex TUI should embody these core principles:

### 1. **Keyboard-First, But Discoverable**
- Power users can navigate entirely via keyboard
- New users can discover features through visual cues
- Contextual help always visible at bottom

### 2. **Instant Feedback**
- Show cached data immediately (optimistic UI)
- Background refresh with visual indicators
- Clear loading states for network operations
- Visible success/error feedback

### 3. **Progressive Disclosure**
- Start simple: just show list of images
- Drill down for more details as needed
- Don't overwhelm with information upfront
- "Overview first, details on demand"

### 4. **Consistent Patterns**
- Same navigation keys work everywhere
- Predictable back/forward flow
- Consistent color coding (blue=selected, yellow=warning, red=error)

### 5. **Graceful Degradation**
- Work offline with cached data
- Handle network errors without crashing
- Provide clear error messages with remediation steps

### 6. **Speed and Responsiveness**
- Never block the UI thread
- 60 FPS rendering
- Responsive to every keystroke

### Responsive Behavior

**Minimum Terminal Size:**
- Width: 60 columns (narrower causes wrapping)
- Height: 15 lines (shorter causes truncation)

**Component Priority** (when space is tight):
1. Title Bar (always)
2. Footer (always)
3. Main Content (always - fills remaining)
4. Breadcrumb (high priority)
5. Status Banner (high priority when present)
6. Status Line (medium priority - hides first)

## Interaction Patterns

### 1. Navigation Consistency

**Back Navigation** (always works the same):
- `â†` / `h` / `Esc` - Go back one level
- Repository List: Quit
- Tag List: Back to repositories
- Details: Back to tags
- Modal: Close modal

**Forward Navigation** (always works the same):
- `â†’` / `l` / `Enter` - Drill down
- On repository: Show tags
- On tag: Show details
- On platform: Select platform

### 2. Search Behavior

**Universal Search Pattern:**
1. Press `/` anywhere
2. Search box gets focus (cursor visible)
3. Type characters (instant filter)
4. Results update in real-time
5. `â†‘` / `â†“` to navigate results
6. `Enter` to select
7. `Esc` to cancel/clear

**Smart Matching:**
- Fuzzy match by default
- Case-insensitive
- Matches anywhere in string
- Highlights matching characters

### 3. Progressive Disclosure

**Three Levels of Detail:**

**Level 1: Overview** (List View)
- Just the essentials
- Name, count, size, date
- Scannable table

**Level 2: Summary** (Tag View)
- More details
- Individual tags
- Platform indicators
- Digests (truncated)

**Level 3: Complete** (Details View)
- Everything available
- Full digests
- Complete config
- All metadata
- Scrollable

### 4. Contextual Actions

Actions change based on context:

**Repository List:**
```
[Enter] View Tags  [/] Search  [r] Registries  [R] Refresh  [q] Quit
```

**Tag List:**
```
[Enter] Inspect  [â†] Back  [/] Search  [R] Refresh  [d] Delete  [q] Quit
```

**Details View:**
```
[â†] Back  [p] Platform  [m] Raw Manifest  [c] Raw Config  [y] Copy  [q] Quit
```

Always contextually relevant, never overwhelming.

### 5. Feedback Loops

**Every Action Gets Feedback:**

- **Keystroke**: Instant visual response (selection moves)
- **Search**: Instant filtering (no lag)
- **Network operation**: Loading indicator appears
- **Completion**: Indicator disappears, data appears
- **Error**: Banner or modal with clear message
- **Success**: Brief confirmation (e.g., "Copied!")

**Feedback Timing:**
- Instant (<50ms): Direct UI changes
- Fast (<200ms): Cached data loads
- Normal (<1s): Network requests with spinner
- Slow (>1s): Progress bar with percentage
- Timeout (>30s): Error with retry option

### 3. Large Data Sets

**Thousands of Repositories:**
- Virtual scrolling (only render visible items)
- Show count: "Showing 1-50 of 5,234"
- Load more as user scrolls
- Search becomes essential

**Huge Manifests:**
- Paginate or truncate in details view
- Offer raw view [m] for complete data
- Horizontal scrolling for wide content

### 4. Invalid States

**Corrupted Cache:**
- Show warning: "Cache error - fetching fresh data"
- Continue with network fetch
- Log error to file

**Registry Removed:**
- Detect on startup
- Show: "Registry 'xyz' no longer configured"
- Default to first available registry

## Performance Targets

### Responsiveness
- **Keystroke to UI update**: <16ms (60 FPS)
- **Search filter update**: <50ms (instant feel)
- **Cache load**: <100ms (feels instant)
- **Network with cache**: Show cached data, refresh background
- **Network without cache**: Show loading spinner, <5s target

### Memory
- **Base**: <10 MB
- **With 1000 repos cached**: <50 MB
- **With 10,000 repos**: <200 MB

### CPU
- **Idle**: <1% CPU
- **Scrolling**: <10% CPU
- **Searching**: <20% CPU (transient)
